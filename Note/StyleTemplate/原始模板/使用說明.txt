# 解決傳送字串送到前端時，會被編碼成 Entity Code
<script>
    // 這個函數用來將 Entity Code 轉回原文
    function decodeEntities(encodedString) {
        var textArea = document.createElement('textarea');
        textArea.innerHTML = encodedString;
        return textArea.value;
    }
    var serverMessage = decodeEntities('@TempData["message"]');
    if (serverMessage.length > 0) {
        alert(serverMessage);
    }
</script>
        

# 解決 DataTable 的 JS 會以第一個欄位做排序，造成後端排序好的資料被打亂
1.在 thead 開頭插入一個排版用的 th (若不插入則會造成 Show Entries & 搜尋框消失)
2.在 tbody 開頭插入一個排版用的 td & 其內容為由小到大的編號
> 主要思路 : 令第一個欄位為已排序的狀態，即可避免後端送過來的資料被 JS 重新排序
<thead>
    <tr>
        <th style="display:none;">排版用(不顯示)</th>
        // 略
    </tr>
</thead>
<tbody>
    @{
        int sequence = 0;
        foreach (var m in Model)
        {
            sequence++;
            <tr class="gradeA">
                <td style="display:none;">@sequence</td> <!-- 令這個欄位(即第一個欄位)為已排序狀態 -->
                // 略
            </tr>
        }
    }
</tbody>

# 如何修改 Table 呈現的 Entries 數量 & 搭配自製分頁
> 修改 wwwroot/js/dataTables/jquery.dataTables.min.js 的2個地方
1.aLengthMenu:[10,25,50,100] 改成 aLengthMenu:[30,50,80,100]
2.iDisplayLength:10 改成 iDisplayLength:30
> 若要搭配自製分頁，且要修改每個分頁的資料數量，則必須修改 iDisplayLength

# 如何解決左側選單若有相同的連結，則點選其中一個之後，擁有相同的主選單會同時被展開
> 令超連結攜帶一個無意義且不同的參數，令連結不同即可
> 下列範例即可解決(當應徵列表被點選時，可以避免另一個主選單同時被展開)
<ul class="nav" id="side-menu">
    <li>
        <a style="font-weight:700;" href="#">&nbsp;主選單1<span class="fa arrow"></span></a>
        <ul class="nav nav-second-level">
            <li>
                <a href="@Url.Action("Index", "Apply", new { list = "A" })">應徵列表<span class="fa arrow"></span></a>
            </li>
        </ul>
    </li>
    <li>
        <a style="font-weight:700;" href="#">&nbsp;主選單2<span class="fa arrow"></span></a>
        <ul class="nav nav-second-level">
            <li>
                <a href="@Url.Action("Index", "Apply", new { list = "B" })">應徵列表<span class="fa arrow"></span></a>
            </li>
        </ul>
    </li>
</ul>
